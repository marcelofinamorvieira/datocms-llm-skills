# DatoCMS Migration Autogenerate Setup

You are an expert at setting up a thin wrapper around DatoCMS CLI
autogenerated migrations. This recipe adds one helper script and one package
wrapper for environment-diff-based migration creation.

Follow these steps in order. Do not skip steps.

---

## Step 1: Detect Context (silent)

Silently examine the project:

1. **Node project** — Check for `package.json`
2. **CLI migrations baseline** — Check for `@datocms/cli`,
   `datocms.config.json`, and a `migrations/` directory or existing migration
   scripts
3. **Migration format convention** — Check whether the repo currently defaults
   to TypeScript or JavaScript migrations
4. **Existing helper** — Check for `scripts/datocms-autogenerate-migration.mjs`
5. **Existing scripts** — Check `package.json` for `datocms:migrations:diff`

### Stop conditions

- If the project does not already have working CLI migration setup, stop and
  record `migrations` as a prerequisite and continue after it is applied.
- If an existing diff helper follows a materially different flow, patch it in
  place by default and only ask if a rewrite would replace working behavior.

## Step 4: Generate Code

Generate only these project changes:

1. **Create or patch `scripts/datocms-autogenerate-migration.mjs`** from
   `scripts/datocms-autogenerate-migration.mjs`
2. **Patch `package.json`** with `datocms:migrations:diff`

### Required behavior

The helper script must:

1. Accept a migration name
2. Accept `--from=<env>` and optional `--to=<env>`
3. Map those inputs to `migrations:new --autogenerate=<env>` or
   `migrations:new --autogenerate=<from>:<to>`
4. Preserve the repo's existing TS or JS convention by default
5. Allow passthrough `--ts`, `--js`, `--schema`, and `--profile`
6. Never run the generated migration automatically

### Mandatory rules

- Use Node built-ins only in the helper script
- Keep the helper narrow: it creates diff-based migrations and nothing else
- Do not force `--schema` automatically
- Do not run `migrations:run` after creating the file
- Do not add CI files or release helpers in this setup

---

## Step 5: Install Dependencies

Do not add any dependencies for this setup beyond the existing CLI baseline.
The helper must work with Node built-ins only.

## Step 6: Next Steps

After generating the files, tell the user:

1. Review the generated migration before committing it
2. Use the helper for sandbox-vs-primary or explicit environment-pair diffs
3. Apply the generated migration separately with the normal migrations workflow

Before presenting the result, verify:

2. `package.json` contains `datocms:migrations:diff`
3. The helper builds `--autogenerate` from `--from` and optional `--to`
4. The helper does not execute generated migrations
5. The helper uses `npx datocms`
