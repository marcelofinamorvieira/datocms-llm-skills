# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

DatoCMS skills repository — 8 public skills that provide focused AI-tool guidance for content delivery, content management, CLI workflows, frontend integrations, plugin development, plugin UI design, plugin scaffolding, and project setup. Ships as static markdown; no build or bundle step.

## Repository Structure

- `skills/<skill-name>/SKILL.md` — skill definition (YAML frontmatter + markdown body)
- `skills/<skill-name>/references/` — detailed reference docs imported by the skill
- `skills/<skill-name>/agents/openai.yaml` — Codex agent interface config, must stay synced with SKILL.md frontmatter
- `skills/datocms-setup/` — special orchestrator skill that routes to 25 internal recipes via `references/recipe-manifest.json`
- `evals/` — trigger evaluation framework (Python scripts, JSON fixtures, result snapshots)
- `docs/` — longer reference material
- `local/` — local-only scratch (gitignored)

## Key Commands

```bash
# Validate repo invariants and metadata sync
python3 evals/scripts/validate_skill_repo.py --repo-root .

# Validate with clean-git gate (pre-publish)
python3 evals/scripts/validate_skill_repo.py --repo-root . --require-clean-git

# Validate with freshness gate for checked-in results
python3 evals/scripts/validate_skill_repo.py --repo-root . --require-fresh-results-sync

# Run Claude Code trigger eval
python3 evals/scripts/run_claude_trigger_eval.py \
  --repo-root . \
  --output-dir evals/results/adHocRuns/<date>-<label>/raw \
  --source combined

# Analyze trigger results
python3 evals/scripts/analyze_trigger_results.py \
  --results-dir <run-dir>/raw \
  --output-json <run-dir>/analysis.json \
  --output-markdown <run-dir>/analysis.md

# Compare baseline vs candidate
python3 evals/scripts/compare_trigger_runs.py \
  --baseline <baseline>/analysis.json \
  --candidate <candidate>/analysis.json \
  --output-markdown <candidate>/comparison.md \
  --output-json <candidate>/comparison.json
```

## Editing Skills

- Each skill's `agents/openai.yaml` has `synced_from_*` comments tracking its source SKILL.md. After changing a skill's frontmatter `name` or `description`, update the corresponding YAML and run validation.
- `datocms-setup` has `disable-model-invocation: true` and `allow_implicit_invocation: false` — it must be invoked explicitly.
- Eval fixtures live at `evals/<skill-name>-skill-eval.json`. Every shipped skill must have a matching fixture.
- When refining trigger boundaries: edit frontmatter `description` first (small deltas), re-run evals before touching the SKILL.md body.

## TypeScript Rules (for generated code)

All code generated by setup recipes must follow `skills/datocms-setup/patterns/MANDATORY_RULES.md`:

- Never use `as unknown as SomeType`
- Avoid `as SomeType` casts — use type guards or fix upstream types
- Prefer `import type { ... }` for type-only imports
- Let TypeScript infer types wherever possible

## Eval Framework

- **Never run evals proactively** — they are expensive (many LLM API calls). Only run when the user explicitly asks.
- Default classification threshold: `0.5` (trigger_rate >= 0.5 = predicted trigger)
- Fixture modes: `implicit`, `explicit`, `overlap` (with `boundary_with` for overlap cases)
- Two eval tracks: Claude Code (`run_claude_trigger_eval.py`) and Codex (`run_codex_trigger_eval.py`)
- Ad hoc runs go in `evals/results/adHocRuns/<date>-<label>/`
- Historical baselines go in `evals/results/historicalRuns/`
